# 第0部：v2.0 設計改訂の要点（なぜこう変えたか）

v1.0の学習設計は正しいが、ゲームとして次の4つの致命的リスクがあった。v2.0はこれらを解決する。

| リスク | 症状 | v2.0の解決策 |
|---|---|---|
| **操作難度の壁** | 格闘ゲームは中学生の大半が「連打マン」になり、フレームの隙を狙う学習設計が機能しない | **アシストモード**（§3.6）：方向+1ボタンのオートコンボ、オートガード。全員を「理（リクツ）」の勝負の土俵に乗せる |
| **GL3安地問題** | 中間ギアが常に無難なら誰もシフトせず、コアの学びが死ぬ | **ギアヒート**（§3.2）：同一ギア連続使用でオーバーヒート。シフトが「たまにやる技」ではなく「常に回す主役」になる |
| **シフト＝純デメリット** | 隙12Fはリスクだけでリターンがなく、シフトが「怖い操作」になる | **パーフェクトシフト**（§3.1）：かみ合いタイミングに合わせた入力で隙を短縮＋強化。リスクに技術介入と快感を与える |
| **ゲームフィールの未定義** | ヒットストップ等の指定がないと、正しく実装しても「安っぽい」動きになる | **ゲームフィール仕様**（§3.8）を新設。数値で規定 |

さらに、**学習ループとゲームループの融合**（クイズ正解が強さに直結する「ちしきゲージ」§5.4）、**授業運用への最適化**（短時間マッチ・教室2P対戦 §3.9・ギア検定 §5.5）、**AIの教習機械化**（三段階エスカレーション §6.5）を追加した。

# 第1部：プロジェクト概要

## 1.1 目的
中学校技術・家庭科（技術分野）「C エネルギー変換の技術」のうち **「回転運動の伝達」「運動を変化させる機構」（教科書 pp.180-185）** を、対戦格闘ゲームのコアメカニクスとして体験的に学習させる。反射神経ではなく **「理（リクツ）＝カリキュラム知識の運用」で勝てる** ゲームデザインとする。

## 1.2 基本情報

| 項目 | 内容 |
|---|---|
| タイトル | GEAR FIGHT |
| ジャンル | 2D対戦格闘（ストリートファイター型）＋パーツ収集カスタマイズ（メダロット型） |
| 主題歌 | NEXT GEAR（ボーカル版 5:12 ／ インスト版 4:46、Suno製、mp3） |
| 対象 | 中学生（技術科の授業・自習用） |
| プラットフォーム | Google Apps Script（GAS）HTMLService で配信する **単一HTMLファイル** |
| 画面 | Canvas **384×216**（16:9）を整数倍スケーリングで全画面フィット |
| グラフィック | GBA風ピクセルアート。**外部画像を使わず fillRect ベースの手続き型スプライト**で全キャラを描画 |
| 操作 | タッチ（バーチャルジョイスティック＋ボタン）／キーボード両対応 |
| UI言語 | 日本語 |
| 通信対戦 | Firebase Realtime Database（フェーズ2）＋ **同一端末2P対戦（教室モード、フェーズ1に格上げ）** |

## 1.3 実装上の制約（必須遵守）
- 単一HTMLファイル（CSS/JSインライン）。GAS HTMLService の `createHtmlOutputFromFile` でそのまま配信可能なこと。
- 外部CDN依存は原則なし。Firebase使用時のみ公式SDKのCDN読込を許可。
- セーブデータ（獲得パーツ、章進行、検定級、勝敗記録）はlocalStorage。`SaveManager` として抽象化し、将来 `google.script.run` 経由のスプレッドシート保存に拡張可能にする。
- スマホ（iPad含む）タッチ操作を第一級市民として設計。`touch-action: none`、ダブルタップズーム抑止。
- 音声はユーザー操作後に再生開始。音源URLは `CONFIG.AUDIO_URLS` で差し替え可能（§8）。
- 1試合の所要時間：**授業モードで最大90秒／試合**を厳守（§3.7）。

# 第2部：学習設計（カリキュラム対応）

## 2.1 教科書内容 → ゲームメカニクス対応表（本作の核）

### A. 回転運動の伝達（教科書 pp.180-181）

| 教科書の概念 | 定義（教科書準拠） | ゲーム内での実装 |
|---|---|---|
| 原動車／従動車 | モータ側の車／伝えられて動く車 | ギアフレンドの「モーター（原動）→腕・脚（従動）」として全機体に内蔵 |
| **速度伝達比** | **従動車の歯数（直径）÷ 原動車の歯数（直径）** | ギア段 GL1〜GL5 の正体。HUDに実際の歯数と比を常時表示 |
| 従動車の歯数が多い→回転は遅く・回転力（トルク）は大きい | 例：原動10枚:従動30枚 → 比3 | 高ギア（GL4-5）＝攻撃力大・動作遅い |
| 従動車の歯数が少ない→回転は速く・回転力は小さい | 例：原動30枚:従動10枚 → 比0.33 | 低ギア（GL1-2）＝速い・一発が軽い |
| 歯車伝動装置：全体の速度伝達比は各段の比の**積** | — | 「段階シフト」システム。GL1→GL5へ一気に飛べず、隣接段へ順に掛け合わせる |
| **歯車のかみ合い** | 歯と歯が正しくかみ合って動力が伝わる | **パーフェクトシフト**（§3.1）：かみ合いの瞬間に合わせるとロスなくシフトできる |
| モーターの連続高負荷 | （発展）高トルク連続運転は発熱する | **ギアヒート**（§3.2）：高ギア連続使用でオーバーヒート |
| 摩擦で伝達（摩擦車・**ベルトとプーリ**）＝滑ることがある | 二軸が離れ回転方向は同じ | ベルト駆動脚：最速だが**オイル・氷ステージでスリップ**する |
| かみ合いで伝達（**歯車・チェーンとスプロケット・ラックとピニオン・ウォームギヤ**）＝滑らない | — | チェーン駆動脚：スリップしない。ラック＝飛び道具射出、ウォームギヤ＝投げ技（大きな回転力） |

### B. 運動を変化させる機構（教科書 pp.182-185）

腕パーツの攻撃モーションは、**実在のリンク・カム機構をそのまま可視化**する。攻撃ヒット時、腕の内部機構がワイヤーフレーム風に一瞬オーバーレイ表示される（学習演出）。

| 機構 | 運動の変換 | 対応する攻撃 |
|---|---|---|
| **往復スライダクランク機構** | 回転運動 → 往復直線運動 | 速度型腕のジャブ連打（ピストンパンチ） |
| **てこクランク機構** | 回転運動 → 揺動運動 | パワー型腕の大振りフック |
| **カム機構（直線）** | 回転運動 → 直線運動（押し上げ） | 対空アッパー |
| **カム機構（揺動）** | 回転運動 → 揺動運動 | 防御型腕のシールドバッシュ |
| **両てこ機構** | 揺動 → 揺動 | ガードモーション（盾の展開） |
| **両クランク機構** | 回転 → 回転 | 回転系必殺技（スピニングアタック） |
| ラックとピニオン | 回転 → 往復直線 | 飛び道具（ボルトシューター）の装填・射出 |
| ウォームギヤ | 直交軸・大きな回転力 | 投げ技（掴んで巻き上げる） |

### C. 機械の共通部品と保守点検（教科書 pp.182-185）

| 教科書の概念 | ゲーム内での実装 |
|---|---|
| 共通部品・JIS規格 | パーツ交換システムの世界観根拠。「ギアポリスの全パーツはJIS規格で統一されているから、どの機体にも組める」とノギ先生が解説 |
| 保守点検 | **パーツ損傷システム（§3.5）とピットイン**：ラウンド間に損傷パーツを1つ応急修理できる。「点検・整備が性能を保つ」の体験化 |

## 2.2 ノギ先生の解説パート
- **ラウンド間リアルタイムヒント（新規）**：ラウンドを落とすと、次ラウンド開始前にノギ先生の**1行ヒント**が表示される（例：「ウィズルは3連打のあと、必ず一瞬止まる。そこだ！」）。敗北するまで解説を待たない。
- 敗北時：「なぜ負けたのか」を物理法則で1〜3枚のメッセージウィンドウで解説。責めない口調。**解説直後に「もう一度たたかう」ボタンで3秒以内に再戦**（ロード・演出スキップ）。
- クリア時：「なぜ勝てたのか」→ 教科書用語との接続 ＋ 確認クイズ3問（正解が次の力になる「ちしきゲージ」§5.4）。

# 第3部：ゲームシステム詳細

## 3.1 ギアシステム（GL1〜GL5）— コア中のコア

各機体は戦闘中いつでもギアシフト可能。**ギア段＝速度伝達比** であり、HUDに歯数モデルと比を表示する。

| ギア段 | 歯数モデル（原動:従動） | 速度伝達比 | 移動速度倍率 | 攻撃発生F | 攻撃ダメージ倍率 | 硬直（全体F） | ガードブレイク | ヒート増加/秒 |
|---|---|---|---|---|---|---|---|---|
| GL1 | 30:10 | 0.33 | ×1.6 | 5F | ×0.5 | 14F | 不可 | −20（冷却） |
| GL2 | 25:15 | 0.6 | ×1.3 | 7F | ×0.75 | 18F | 不可 | −10（冷却） |
| GL3 | 20:20 | 1.0 | ×1.0 | 10F | ×1.0 | 24F | 不可 | ±0 |
| GL4 | 15:25 | 1.67 | ×0.8 | 15F | ×1.5 | 34F | **可** | +15 |
| GL5 | 10:30 | 3.0 | ×0.6 | 20F | ×2.2 | 44F | **可（削り大）** | +30 |

（60fps基準。数値は初期バランスであり定数テーブル `GEAR_TABLE` に一元管理し調整可能にする）

### シフト仕様と【パーフェクトシフト】（v2.0 新規・最重要システム）
- **隣接段への段階シフトのみ**（飛び級不可）。「歯車伝動装置は各段の比の積」の体現。
- 通常シフト：所要**12F、この間ガード不能・無防備**（第2章の学習ポイント）。
- **パーフェクトシフト**：シフト開始時、HUDの歯車インジケータが回転し、歯の「かみ合い位置」が周期的に光る（周期30F、判定窓±4F）。**光の瞬間にもう一度シフトボタンを押すと、シフト所要が4Fに短縮**され、機体が一瞬発光、次の1発の攻撃にダメージ+20%が乗る。
  - 学習的意味：「歯車は歯が正しくかみ合う瞬間に動力が伝わる」を指と目で覚える。
  - ゲーム的意味：シフトが「怖い操作」から「決めると気持ちいい操作」へ。上級者の技術介入点。
  - 効果音：通常シフト＝ガチャン／パーフェクト＝キィン（高音）＋ヒットストップ2F。
- シフト中に被弾すると被ダメージ×1.5＋相手に **THEORY BONUS**。
- シフト入力は**先行入力バッファ8F**を持つ（硬直中に入力しておくと硬直明けに自動発動）。

## 3.2 ギアヒートシステム（v2.0 新規）

「GL3に居座れば安全」を構造的に潰し、**シフトを常に回す**ためのシステム。実際のモーターの発熱と同じ理屈。

- ヒートゲージ 0〜100。GL4/5使用中に増加、GL1/2で冷却（増減値は上表）。攻撃ヒット時はさらに+5/発。
- **100到達＝オーバーヒート**：3秒間、強制GL1固定＋機体から蒸気エフェクト＋移動以外の性能▲30%。
- HUDのヒートゲージは60超で黄色、85超で赤点滅＋警告音。
- 学習的意味：「大きなトルクを出し続けると機械は熱を持つ。冷却（低負荷運転）が必要」。
- 結果として理想の戦い方が「低ギアで立ち回り→好機に高ギアへ段階シフト→仕留めて冷却」という**教科書どおりのギア運用のリズム**になる。

## 3.3 基本アクション

| 操作 | 内容 |
|---|---|
| 移動 | 歩き／後ろ歩き（＝ガード方向）／ダッシュ（GL依存速度） |
| ジャンプ | 前・垂直・後ろ |
| 弱攻撃（A） | 発生の速い攻撃。現GLの発生F・ダメージ倍率を適用 |
| 強攻撃（B） | 大ダメージ。GL4-5でのみガードブレイク属性 |
| 飛び道具（C） | ボルトシューター（ラック＆ピニオン機構）。弾速はGLに反比例、弾威力はGLに比例 |
| 投げ（近距離でA+B同時） | ウォームギヤ・ワインダー。ガード不能、ダメージ固定150 |
| ガード（後ろ入力） | 削りダメージあり。ガード耐久ゲージ0でガードクラッシュ |
| ギアシフトUP/DOWN（専用ボタンL/R） | §3.1参照 |
| 超必殺技（ゲージ100%で専用ボタン発光→タップ） | 装備パーツの組み合わせで技が変化（§3.5）。A+B+C同時押しはタッチで困難なため**専用ボタン方式**に変更 |

- 体力：両者1000。超必ゲージ：与ダメ・被ダメ・THEORY BONUSで増加。
- **快適性仕様（必須）**：全攻撃に先行入力バッファ6F／ジャンプ踏み切りにコヨーテタイム3F／振り向きは自動。

### THEORY BONUS（セオリーボーナス）
「教科書どおりの正しい判断」をした瞬間に発生する報酬演出。
- 発生条件：①相手のシフト隙に攻撃を当てる ②GL4以上でガードブレイクを決める ③地形有利脚で地形ギミックを踏み越えて攻撃を当てる ④相手の飛び道具を低ギアダッシュで潜って反撃 ⑤相手のオーバーヒート中にコンボを決める
- 効果：「THEORY BONUS!」＋該当する教科書用語（例「シフト中は無防備！」）がカットイン。超必ゲージ+25%、獲得GP+。
- **リザルトでTHEORY BONUSの内訳を表彰**（「今日の理論家ポイント」）。ダメージ量ではなく理解の質を褒める。

## 3.4 地形・環境システム（第3章の学習テーマ）

### 地面属性
| 属性 | 効果（脚相性なしの場合） | 有利な脚 |
|---|---|---|
| 通常（舗装） | なし | すべて |
| 氷 | 停止時に8Fの滑り、ダッシュ後慣性1.4倍 | スパイク脚（氷用） |
| 砂 | 移動速度×0.75、ジャンプ高さ×0.9 | ワイドソール脚（砂用）、キャタピラ脚 |
| 段差あり | ステージ中央に高低差 | 高速脚（ジャンプ強化） |
| オイル床（部分） | **ベルト駆動脚のみスリップ**（摩擦伝達は滑る！） | チェーン駆動脚 |

### 環境イベント（ランダム／ステージ固有）
- 野良犬乱入：一定時間ごとに画面を横切り、接触した側が6Fよろけ。ジャンプで回避可。**接近1秒前に画面端に「!」予告**（理不尽さの排除。予告があれば読み合いの材料になる）。
- オイルこぼれ：ステージの一部が一定時間オイル床化（こぼれる前にドラム缶が転がってくる予告演出）。
- 天候（雨）：全地面の摩擦係数低下。

## 3.5 パーツカスタマイズ＋パーツ損傷（メダロット方式）

### パーツスロット：頭・右腕・左腕・脚 の4スロット
- **頭**：HP補正・特性（センサーヘッド＝相手のGLとヒート可視化、アーマーヘッド＝HP+100）
- **腕**：攻撃タイプを決定。左右で別タイプ装備可
- **脚**：移動性能と地形適性・駆動方式を決定

### 腕パーツ一覧（シルエット仕様は設定書の表に準拠）
| パーツ | 機構 | 得意GL | シルエット | 質感（パレット） |
|---|---|---|---|---|
| スライダアーム（速度型） | 往復スライダクランク | GL1-2 | 細く長い・先端が尖る | 軽金属・光沢（明るいシルバー＋赤） |
| ガンロックアーム（パワー型） | てこクランク | GL4-5 | 太く短い・拳が肥大 | 鈍い鉄色（ダークグレー） |
| イージスアーム（防御型） | カム（揺動） | GL3-4 | 盾状装甲が肘と拳を覆う | マット装甲板（紺＋白） |

### 脚パーツ一覧
| パーツ | 駆動方式 | 特徴 |
|---|---|---|
| ホイールレッグ（高速型） | **ベルト駆動** | 最速。ただしオイル・氷でスリップ |
| クローラーレッグ（重装型） | キャタピラ | 遅いが被弾よろけ耐性・砂に強い |
| スパイクレッグ（氷特化） | チェーン駆動 | 氷でノーペナルティ |
| ワイドレッグ（砂特化） | チェーン駆動 | 砂でノーペナルティ |

### パーツ損傷システム（v2.0 新規・メダロットの魂）
- 腕・脚パーツはそれぞれ装甲値200を持つ。**被弾部位はランダムではなく攻撃の高さで決定**（上段攻撃→腕、下段攻撃→脚）。
- 装甲値0で**パーツブレイク**：外装が弾け飛ぶ演出（破片パーティクル）＋そのパーツの性能▲40%（腕＝攻撃力低下、脚＝移動低下）。見た目もフレームむき出しに変化。
- **ピットイン**：ラウンド間の5秒間に、損傷パーツ1つを選んで応急修理（装甲50%回復）できる。教科書「保守点検」の体験化。
- ゲーム的意味：「どこを狙うか／どこを守るか」の新しい読み合い。速度型脚を折れば地形有利が消える、など戦略が生まれる。

### タイプ相性（ポケモン式、ダメージ±20%）
組み合わせで機体タイプが決まる：**スピード → パワー → ディフェンス → スピード**（三すくみ）。腕2本の型が一致でそのタイプ、混成なら「バランス」タイプ（相性なし）。

### 超必殺技（腕の組み合わせで決定）
| 右腕×左腕 | 超必 | 演出 |
|---|---|---|
| 速度×速度 | マッハ・ピストンラッシュ | スライダクランク高速連打→フィニッシュ直線突き |
| パワー×パワー | ギガント・スイング | 両てこクランクの揺動大回転 |

## 3.6 アシストモード（v2.0 新規・アクセシビリティの生命線）

## 6.2 描画仕様

- 攻撃ヒット時：機構ワイヤーフレームの1秒オーバーレイ（例：ジャブ時に往復スライダクランクの回転→直線変換アニメを腕の上に半透明表示）。
- アニメーション：各ポーズ2〜4コマで十分（GBA風）。コマ数より§3.8のゲームフィールを優先。

- 情報過多を防ぐため、③の数式表示は設定でOFF可（慣れた生徒向け）。

## 6.4 確認クイズ問題バンク（章別・3択・正答は★）

**第1章（速度伝達比の基本）**
1. 速度伝達比の求め方は？ → ★従動車の歯数÷原動車の歯数／原動÷従動／歯数の和
2. 原動車10枚・従動車30枚のとき速度伝達比は？ → ★3／0.33／300
3. 従動車の歯数が少ないと回転速度は？ → ★速くなる／遅くなる／変わらない
4. 従動車の歯数が多いと回転力（トルク）は？ → ★大きくなる／小さくなる／変わらない
5. ウィズルの連打が軽い理由は？ → ★速度伝達比が小さくトルクが小さいから／軽い素材だから／腕が短いから

**第2章（トルクと隙）**
1. 原動30枚・従動10枚の速度伝達比は？ → ★0.33／3／20
2. 複数の歯車を組み合わせた装置を何という？ → ★歯車伝動装置／摩擦車装置／カム装置
3. 歯車伝動装置全体の速度伝達比は各段の比の？ → ★積／和／差
4. モータの回転を直接伝える車を何という？ → ★原動車／従動車／遊び車
5. ガンロックのシフト中に攻撃が通る理由は？ → ★かみ合いが切り替わる間は力を伝達できないから／装甲が薄いから／目を閉じているから

**第3章（伝達方式と摩擦）**
1. 二軸が離れていて回転方向が同じになる摩擦伝達は？ → ★ベルトとプーリ／歯車／ラックとピニオン
2. 二軸が離れていて回転方向が同じ、かつ滑らない伝達は？ → ★チェーンとスプロケット／摩擦車／ベルト
3. 回転運動を往復直線運動に変換するのは？ → ★ラックとピニオン／ウォームギヤ／プーリ
4. 二軸が直交し大きな回転力を得られるのは？ → ★ウォームギヤ／摩擦車／チェーン
5. オイル床でホイールレッグ（ベルト駆動）が滑る理由は？ → ★摩擦で動力を伝えているから／重いから／歯数が多いから

**第4章（機構と部品）**
1. 回転運動を揺動運動に変える4節リンクは？ → ★てこクランク機構／両クランク機構／往復スライダクランク機構
2. 回転→直線（押し上げ）に変える機構は？ → ★カム機構／両てこ機構／てこクランク機構
3. 回転→往復直線に変えるリンク機構は？ → ★往復スライダクランク機構／両てこ機構／両クランク機構
4. 機械で共通して使える部品は何に基づいて作られる？ → ★JISなどの規格／メーカー独自基準／職人の勘
5. アイギスのガードを崩すには？ → ★GL4以上の高トルク攻撃／連打／投げ以外不可能

**第5章（統合）**
1. 揺動→揺動の機構は？ → ★両てこ機構／両クランク機構／カム機構
2. 回転→回転の機構は？ → ★両クランク機構／両てこ機構／スライダクランク
3. 速度とトルクの関係は？ → ★反比例的なトレードオフ／比例／無関係
4. GL2(比0.6)からGL4(比1.67)へ2段シフトしたとき、考え方として正しいのは？ → ★各段の比を順に掛け合わせて変化する／一瞬で切り替わる／比は足し算される
5. 対戦前に相手のパーツ構成を知ることの利点は？ → ★相性と対策を事前に設計できる／攻撃力が上がる／利点はない

## 6.5 AIプロファイル＋三段階エスカレーション（v2.0 改訂）

全ライバルAIは試合中に3段階で「本気度」が上がる。**プレイヤーが優勢のときだけ**次段階へ進む（劣勢の生徒には牙をむかない＝ラバーバンド）。常に「あと少しで勝てた！」の体験を作る。

```js
AI_PROFILES = {
  kakashi: { stages:[{move:0},{move:0.3},{guardRate:0.8}] },  // チュートリアル用
  sonica:  { glRange:[1,2], aggression:0.9, comboLen:3, gapAfterCombo:12/*F*/,
             stages:[{gap:16},{gap:12},{gap:10, feintRate:0.2}] },
  gouken:  { glRange:[4,5], shiftFreq:0.5, shiftTelegraph:true, heatCareless:true,
             stages:[{telegraphGlow:1.5},{telegraphGlow:1.0},{mixGL3:0.3}] },
  rin:     { glRange:[2,4], stage:'ice', zoning:0.7, projectileFreq:0.5,
             stages:[{zoning:0.5},{zoning:0.7},{terrainTrap:true}] },
  kamei:   { glRange:[3,4], aggression:0.15, guardRate:0.9, counterOnly:true,
             stages:[{guardRate:0.8},{guardRate:0.9},{counterAfterGuard:0.5}] },
  kai:     { glRange:[1,5], adaptive:true, revealParts:true, superFreq:0.6,
             perfectShiftRate:0.5 },
  omeganova:{ glRange:[1,5], modeSwitch:['speed','power','terrain','guard'],
             switchInterval:600/*F*/, perfectShiftRate:0.7 },
  nogi:    { glRange:[1,5], adaptive:true, reactionF:4, damageBoost:1.3,
             perfectShiftRate:1.0, heatPerfect:true, akuma:true }
}
```

- 共通ルール：AIは**人間に可能な反応速度**（最速でもreactionF:4）を守る。読み合いに勝つAIであって、超反応するAIにしない。

## 6.6 通信対戦（フェーズ2：Firebase）
- Firebase Realtime Database。ルームコード方式（4桁）。
- 同期方式：入力同期（各フレームの入力のみ送信、両端で決定論的シミュレーション）。乱数はルーム共有シード。
- 遅延対策：4F入力ディレイ固定（初版はロールバック不要）。
- Firebase設定は `CONFIG.firebase` に外出しし、未設定時は通信対戦メニューを非表示。
- **注**：教室内の対人戦は §3.9 の同一端末2Pで先行提供する。Firebaseは家庭学習・クラス間対戦向け。
