# ギアフレンド 設計メモ（新エンジン用スプライト仕様 ＋ キャラ性格づけ）

このドキュメントは、作り直した箱ベース戦闘エンジン（`src/combat/`）の上に載せる
**キャラクター「ギアフレンド」**の設計指針。目的は2つ：
1. **どんなスプライトが必要か**を、エンジンの状態から逆算して確定する（作画・生成の発注書）。
2. 各キャラの**動きの個性を「ギア／クランク＝動力の伝達」の物理に結びつける**（＝本作の差別化）。

> 大原則：作画は **右向き基準・足元原点** で1セット作る。エンジンが2P側を自動ミラーする
> （`src/combat/types.ts` の設計法則）。左向きの絵は一切要らない。過去の「向き不整合」事故は
> これで構造的に起きない。

---

## 1. 必要なスプライト（ポーズ）一覧 ＝ エンジン状態からの逆算

エンジンの状態（`FighterPhase`）と技（`moves.ts`）から、各ギアフレンドに必要なポーズは以下。
括弧内は推奨フレーム数（アニメの枚数）。

### 基本ステート
| ポーズ | 用途 | 枚数目安 |
|---|---|---|
| `idle` | 待機（微呼吸） | 2–4 |
| `walk` | 前後歩行サイクル | 4（脚交互＋上下バウンス） |
| `crouch` | しゃがみ | 1 |
| `jump` | ジャンプ（上昇→頂点→下降で差分あると映える） | 1–3 |
| `block` / `crouchblock` | 立ち／しゃがみガード | 各1 |
| `hitstun` | 被弾のけ反り | 1–2 |
| `knockdown` | ダウン（倒れ） | 1–2 |
| `wakeup` | 起き上がり（起き攻めの読み合いで重要な"間"） | 1–2 |
| `dizzy` | 気絶（ふらつき＋頭上に★は演出側で付与） | 2 |
| `victory` / `defeat` | 決着ポーズ | 各1 |
| **`overheat`** | **★新規：オーバーヒート（火花／赤熱／煙）** | 2 |

### 技（攻撃）ポーズ
| 技 | 機構テーマ | ポーズの肝 | 枚数 |
|---|---|---|---|
| `standLight` 立ち弱 | 往復スライダクランク | まっすぐ前に**踏み込んで突く** | 2–3 |
| `standHeavy` 立ち強 | てこクランク | 体重を乗せた**大振り** | 3 |
| `crouchLight` しゃがみ弱 | 小型スライダ | 低い姿勢の**下段差し込み** | 2 |
| `crouchHeavy` 足払い | 回転カム | **低く薙ぐ**（ダウン奪取） | 3 |
| `jumpLight` / `jumpHeavy` 空中攻撃 | — | **斜め下への攻撃**（飛び込み＝中段） | 各2 |
| `fireball` 波動ギア | 発射機構 | **前に押し出す**（飛び道具生成の溜め→放出） | 3＋弾 |
| `dpunch` 昇龍ギア | 打ち上げリンク | **真上に伸びる**無敵対空 | 3 |
| `super` ギアマックス | 全開 | **金色・全身のギアが唸る**大技 | 4–6＋フラッシュ |
| `throw` 投げ | 鋏／万力（クランプ） | **掴んで固定**（一方向に閉じる） | 2 |

**合計：1キャラあたり実質 30〜40 コマ**（サイクルの中割り含む）。過去のAI生成シートと
同じボリューム感なので、1キャラ1枚の大シートで生成→切り出しが現実的。

## 2. 各ギアフレンドの「動きの個性」＝ギア／クランクで差別化

キャラの機体タイプ（`config/parts.ts` の速度／パワー／防御／バランス）と、**得意ギア帯**を
動きに反映する。**同じ技でも機体で"間合い・弧・重み"が変わる**のが本作の肝。

| ギアフレンド（機体） | パイロット | タイプ | 動きの個性（ギア/クランク観点） | 映えポイント |
|---|---|---|---|---|
| **ハジメくん** | 主人公 | バランス | 教科書どおり。全ギアを素直に使える基準機 | 各機構が一番"見える"素直な作画 |
| **ウィズル** | ソニカ | 速度 | 低ギア偏重・手数と差し込み。往復スライダクランクが高速 | 残像・高RPMの軽快さ |
| **ガンロック** | ゴウケン | パワー | 高ギア偏重・一撃重量級。てこクランクの大振り | 溜め→解放の重量・地響き |
| **アイギス** | カメイ | 防御 | ガード・カウンター寄り。低ギアで固く受ける | 盾・装甲・受け流しの火花 |
| **ドリフト** | リン | 速度 | 機動・めくり。空中クランクで弧を描く飛び込み | 迷彩グリーンの軽やかな旋回 |
| **テオリオン** | カイ | バランス（技巧） | 飛び道具・設置・時間差カム | 幾何学的・理知的なエフェクト |
| **オメガノヴァ** | レイ（ラスボス） | 全能 | 全ギア高水準・オーバーヒートしにくい | 黒/赤/金の威圧・一回り大きい |
| **ソフィス・レギオン** | ノギ（隠しボス） | 変則 | 予測不能な機構切替 | 白/銀の異質な多重機構 |

> ※ パイロット名＝人物、機体名＝ギアフレンド（例：ソニカが乗るのがウィズル）。既存の
> ストーリー設定（`docs/仕様書.md`・オープニング会話）をそのまま流用。

## 3. グラフィックが"映える"方向性

- **機体は明確にメカ**（ロボ格闘）。関節に**歯車・クランク・ロッドが覗く**デザインにすると、
  攻撃時の機構オーバーレイ（`往復スライダ/てこ/カム/クランプ`）と絵が一致して説得力が出る。
- **ギア帯で見た目が変わる**と最高：低ギア＝軽装・青系グロー、高ギア＝装甲展開・赤熱。
  最低でも**オーバーヒート時に赤熱＋火花＋煙**の差分は入れたい（教育的にも"熱の限界"が伝わる）。
- **ヒット時の機構ワイヤーフレーム演出**（旧実装の資産）を新エンジンにも移植すると、
  「回転→直線／揺動」の違いが遊びながら身につく＝**本作にしかない画**になる。
- 解像度はレトロ縛りを外してよい（ユーザー方針）。**きれいなドット〜セミHD**で、
  金属のハイライトとギアの陰影が出る塗りが映える。

## 4. 制作フロー（新エンジンへの載せ方）

1. 本仕様の**ポーズ一覧**でAI生成プロンプトを作る（`docs/sprite_generation_prompts.md` を新エンジン
   のポーズ表に合わせて更新）。**右向き・緑背景・足元そろえ・文字なし**は厳守。
2. 生成→緑背景切り出し→**足元原点で配置**（エンジンの箱と一致）。
3. 描画レイヤー（現在の箱表示 `drawFighter`）を、箱の上にスプライトを重ねる方式に拡張。
   当たり判定は箱のまま（見た目と判定を分離）。
4. まず**ハジメくん1体**を通しで載せて、全ポーズ・全技がスプライトで破綻しないか検証してから
   横展開する。

> 現状は幾何学プレースホルダで戦闘の"手触り"を固めている段階。手触りが鉄心の合格ラインに
> 達したら、この仕様に沿ってスプライトを1体ずつ載せる。
